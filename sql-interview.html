SQL Interview: Must Know Queries

Top database and SQL query questions focusing on analytics, aggregations, and edge cases.

1. How do you find the Nth highest salary from an Employee table?

There are a few ways to solve this, but the most robust method in modern SQL is using the DENSE_RANK() window function, which handles duplicates automatically.

WITH RankedSalaries AS (
    SELECT 
        salary, 
        DENSE_RANK() OVER (ORDER BY salary DESC) as rnk
    FROM Employee
)
SELECT salary 
FROM RankedSalaries 
WHERE rnk = N; -- Replace N with the desired rank (e.g., 2 for second highest)

Alternatively, using LIMIT and OFFSET (MySQL/PostgreSQL):

SELECT DISTINCT salary
FROM Employee
ORDER BY salary DESC
LIMIT 1 OFFSET N-1;
Tricky Part: You must account for duplicates! If two employees share the top salary, the 2nd highest salary should be the next distinct value. Using DENSE_RANK() or DISTINCT is mandatory. Using RANK() will skip ranks if there are ties.
2. What is the difference between WHERE and HAVING clauses?

WHERE is used to filter rows before any grouping occurs. It cannot be used with aggregate functions like SUM() or COUNT().

HAVING is used to filter groups after the GROUP BY clause has been applied. It is specifically designed to be used with aggregate functions.

SELECT department_id, SUM(salary) as total_salary
FROM Employee
WHERE status = 'ACTIVE' -- Filters individual employees FIRST
GROUP BY department_id
HAVING SUM(salary) > 500000; -- Filters grouped departments LATER
Tricky Part: Interviewers will ask if you can use both in the same query. Yes, you can! Always filter as much as possible with WHERE first, because reducing the row count before grouping is significantly faster.
3. Explain the difference between INNER JOIN, LEFT JOIN, and FULL OUTER JOIN.

INNER JOIN returns only the rows where there is a match in both tables.
LEFT JOIN returns ALL rows from the left table, and the matched rows from the right table. If there is no match, the right side will contain NULLs.
FULL OUTER JOIN returns ALL rows when there is a match in either the left or right table. Unmatched rows on either side will contain NULLs for the other side.

Tricky Part: If you are filtering on the right table of a LEFT JOIN using the WHERE clause, it implicitly converts the LEFT JOIN into an INNER JOIN by discarding the NULLs! Put right-table conditions in the ON clause instead.
